# Чек-лист качества: лендинг

> Использует `landing-architect` перед отдачей структуры Повелительнице. Если хотя бы один пункт из «обязательных» проваливается — переписываем.
> Опирается на [tone-of-voice.md](../knowledge-base/tone-of-voice.md), [audience.md](../knowledge-base/audience.md), [zerocoder-facts.md](../knowledge-base/zerocoder-facts.md).

---

## Обязательные (без них не отдавать)

- [ ] **Есть опорный референс** — присланный Повелительницей или явно согласованный. Без референса работать нельзя.
- [ ] **Первый экран — крючок-обещание под боль ЦА**, не «о нас», не «приветствуем».
- [ ] **Блок «Для кого»** — конкретные ролевые сегменты с описанием ситуации (см. [audience.md](../knowledge-base/audience.md)), не «для всех, кто интересуется ИИ».
- [ ] **Логика прокрутки прописана** — каждый блок имеет «назначение» и «логику перехода к следующему».
- [ ] **CTA дублируется минимум 3 раза** — на первом экране, после оффера, в финальном блоке.
- [ ] **Продукт описан правдиво** — все продукты, цены, тарифы, гарантии — из [zerocoder-facts.md](../knowledge-base/zerocoder-facts.md), не выдуманные.
- [ ] **Гарантия возврата 14 дней** — упомянута в офферном блоке.
- [ ] **Доступность** — если продукт про российские нейросети, фраза «без VPN, без иностранных карт» есть в видимом месте.
- [ ] **Тон по ЦА** — соответствует целевому сегменту (см. [tone-of-voice.md](../knowledge-base/tone-of-voice.md) § различия по сегментам).

## Желательные

- [ ] **Социальные доказательства** идут после оффера, не до. Сначала обещание, потом подтверждение.
- [ ] **Закрыто 3+ возражения** конкретного сегмента в теле лендинга.
- [ ] **Цифры из правды** — выпускники (16 000), оценка (4.5), аккредитация Сколково, лицензия ДПО.
- [ ] **Кейсы конкретные** — с деталями (профессия + ситуация + результат), не абстрактные «многие из наших выпускников».
- [ ] **Промпт для главного визуала** собран через скилл `Image-Prompt`.
- [ ] **Структура референса прочитана и адаптирована**, а не скопирована вслепую. Где отошли — отмечено почему.

## Стоп-сигналы (если есть хотя бы один — переписать)

- [ ] Первый экран «Добро пожаловать», «О школе», «Приветствуем».
- [ ] Блок «Для кого: все, кто хочет освоить ИИ» — это значит «ни для кого».
- [ ] Дублирование смыслов между блоками («о программе» = «что внутри» = «чему научитесь»).
- [ ] Блок, у которого нет назначения — «для красоты», «потому что в референсе тоже было». Удалить.
- [ ] Цифры без источника («99% доходимости», «10 000 успешных проектов»).
- [ ] Партнёрства, которых у Zerocoder нет (см. [zerocoder-facts.md](../knowledge-base/zerocoder-facts.md) § 5 — партнёрств с крупными компаниями нет).
- [ ] Упоминание акселератора или буткемпа как продаваемых сейчас — они в разработке.
- [ ] CTA «Узнайте больше», «Оставьте заявку» — переформулировать на конкретное действие.

## Формат самопроверки

1. Пройти весь markdown сверху вниз глазами Повелительницы: «есть ли тут блок, который я бы выкинула?» — если да, переделать.
2. Прочитать как ЦА: «после какого блока я бы хотел купить?» — если такого момента нет, лендинг не работает.
3. Проверить — есть ли в каждом блоке либо умножение желания купить, либо закрытие возражения. Если нет — блок лишний.
