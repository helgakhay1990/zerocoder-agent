# Чек-лист ТЗ на монтаж вебинара

Инструкция для агента: при постановке задачи на монтаж эфира — идти по этому списку, фиксировать тайминги, оформлять ТЗ монтажёру.

---

## Что вырезаем (полностью убираем фрагмент)

1. **Временные маркеры в речи:**
   - Упоминания конкретных дат и дней недели.
   - Названия месяцев и времён года («сейчас зима», «летом», «в марте»).
   - Сезонные события и праздники (масленица, НГ, 8 марта, чёрная пятница, распродажи).
   - Привязка к текущим новостям («вот вчера такое-то событие»).

2. **Цены и старые офферы:**
   - Любые озвученные старые цены курсов и доп.продуктов.
   - Старые упоминания скидок, акций, рассрочек, которые уже не актуальны.
   - Описания офферов, которые сняты с продаж (бесплатный доступ к Перплексити, ChatGPT-4 с регистрацией, ВПН-инструкции, устаревший состав нейроабонемента и т. п.).
   - Слайды со старыми ценами — заменить на новые (отметить тайминг + указать, какой слайд ставим).

3. **Речевой мусор:**
   - Кашли, чихи, икота.
   - Длинные «эээ», «мэканья», паузы-затупы.
   - Сквернословие и ругательства.

   > ⚠️ **Честная граница:** whisper кашли/«эээ»/затупы НЕ размечает (это аудио, не лексика). Агент по транскрипту их не найдёт. Поэтому НЕ выдавать список речевого мусора как «вычищено» — это вводит в заблуждение. Правильно: либо отдельным проходом по аудио (если делаем), либо явно вынести монтажёру: «речевой мусор (кашли/эээ/затупы) — почистить на слух по этому правилу». Не делать вид, что поймали всё из текста.

4. **Технические косяки:**
   - Зависания, лаги, эхо, обрыв звука.
   - «У меня не открывается», переоткрытия, переподключения.
   - Фразы «это вторая попытка веба», «давайте перезапустим».
   - В начале эфира — затупы и тех.моменты (часто первые 5–15 минут).

---

## Что заблюриваем (фрагмент остаётся, но картинка прикрывается)

- Даты и время, видимые на экране в практической части:
  - Терминалы, IDE.
  - Чат-боты (Сейлбот и др.) — обычно справа вверху.
  - Интерфейсы инструментов и сервисов.
  - Календари, расписания, дашборды.
- Любая «живая» демонстрация, где случайно мелькает текущая дата/время.

> ⚠️ **Вывод «блюр не требуется» — только после реального визуального прохода** практической части (`visual-scanner` по кадрам, плотный шаг по демо-сегментам), а не по умолчанию. Даты на экране и личные данные транскрипт не видит — их ловят только глазами по кадрам. Если кадры демо не отсмотрены — писать «практику на блюр не проверял», а не «блюр не нужен».

> 🔎 **OCR-проход для исчерпывающего списка дат.** Шаг отсмотра 30 сек мал для коротких событий: дата может мелькнуть в интерфейсе на 5–10 секунд между кадрами выборки и потеряться. По демо-сегментам гнать OCR-проход — нарезка кадров шагом 5 сек через ffmpeg → `tesseract` по каждому → grep дат по распознанному тексту. На выходе — полный перечень таймкодов с датами на экране, без пропусков. Без него ТЗ не даёт монтажёру исчерпывающий список, и он вынужден искать сам — это против правила «монтажёр исполняет, а не ищет». OCR даёт полноту списка, `visual-scanner` — квалификацию (это интерфейс инструмента, а не слайд-нарратив). Подтверждено боевым прогоном OpenClaw 2026-06-03.

---

## Навигация: блок взаимодействия с залом

Не «вырезать» и не «блюр» — отдельный навигационный слой. Монтажёру (и Ольге) нужно быстро найти, **где спикер работает с аудиторией**: открывающий прогрев и закрывающий Q&A часто двигают, сокращают или выносят отдельно.

Сигналы в речи: вопросы к залу («как вы думаете», «кто из вас»), просьбы реакций («напишите в чат», «поставьте плюс/единичку», «с какого города»), переклички, «давайте вопросы». Ловятся транскриптом (`montage_tz.py` → раздел «Взаимодействие с аудиторией»; категория 8 у `transcript-analyzer`).

В ТЗ давать **границами блоков** (начало–конец), а не россыпью реплик: «прогрев `0:00–0:01`», «Q&A `2:25–конец`». Если автор прямо просит найти такой блок — это первоочередная находка, не сноска.

## Формат итогового ТЗ для монтажёра

Сохраняем в Google-таблицу или Google-документ. Структура:

| Поле | Что заполняем |
|---|---|
| **Запись** | Ссылка на эфир (Kinescope / university.zerocoder.ru) |
| **Презентация** | Ссылка на актуальную презу с новыми слайдами (если нужна замена) |
| **Тайминг** | ММ:СС или диапазон ММ:СС–ММ:СС |
| **Действие** | `вырезать` / `заблюрить` / `заменить слайд` |
| **Что именно** | Краткое описание: «упоминание цены 39 900», «затуп», «дата в терминале», «слайд с тарифом ВИП — заменить на новый» |
| **Комментарий** | При необходимости — детали для монтажёра |

## Алгоритм работы агента

1. Получить ссылку на запись эфира.
2. Транскрибировать (или взять готовый транскрипт, если есть).
3. Пройти по транскрипту с этим чек-листом, выписать тайминги. **Границы реза — по пословным таймкодам из whisper `.json`** (там тайминг по словам), а не «на глаз ±1 сек»: давать точную точку начала/конца фразы, чтобы Ольге не приходилось уточнять, где резать.
4. Для практической части — НЕ ограничиваться транскриптом: прогнать `visual-scanner` по кадрам демо-сегментов (даты на экране, личные данные, чужие вкладки). Отметить тайминги на заблюр или явно «не отсмотрено».
4а. **Речевой мусор** (кашли/эээ/затупы) — транскрипт не покажет; вынести отдельным правилом монтажёру (см. ⚠️ выше), не выдавать неполный список как полный.
4б. **OCR-проход по демо-сегментам** для исчерпывающего списка дат на экране (ffmpeg нарезка 5 сек → `tesseract` → grep дат, см. 🔎 выше). Шаг 30 сек пропускает короткие мельканья — OCR закрывает пропуски. Это операционный шаг главного оркестратора. Список дат от OCR → квалификация контекста через `visual-scanner` → тайминги в таблицу на заблюр/замену.
5. Собрать таблицу в формате выше.
6. Если есть сомнения по конкретному моменту — отметить отдельно с пометкой «уточнить у Ольги», не вырезать самовольно.

---

## Чего не делаем без подтверждения

- Не вырезаем содержательные куски эфира (объяснения, разборы), даже если кажется, что они слабые.
- Не вырезаем продающую часть целиком — только перечисленные выше элементы внутри неё.
- Если непонятно, актуальна ли цена/оффер — спрашиваем, не угадываем.
