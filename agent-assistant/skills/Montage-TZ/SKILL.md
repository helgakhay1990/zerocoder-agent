# Montage-TZ

Быстрый черновик ТЗ на монтаж любой записи эфира: транскрипт по всей записи + OCR по окнам шеринга экрана → размеченный список кандидатов с таймкодами.

Это не про решения «что именно вырезать» (это `video-edit-assistant` и автор) и не про содержание эфира (`methodist`). Это операционный слой: за минуты собрать **исчерпывающий список кандидатов**, чтобы человек не отсматривал запись вручную весь вечер.

Полный дизайн ветки — `docs/2026-06-04_montage-tz-skill_doc.md`.

---

## Когда использовать скилл

Активировать, когда задача связана с:
- Подготовкой ТЗ на монтаж записи эфира для монтажёра.
- Поиском в записи дат/секретов на экране, устаревших цен/офферов и дат в речи, тех. заминок.
- Любой записью эфира (вебинар, бизнес-разбор, модуль интенсива, промо-ролик) — инструмент общий, не под конкретный эфир.

**Не использовать для:** написания анонсов (`announce-writer`), структуры слайдов (`methodist`), концепции запуска (`launch-strategist`), операционной подготовки к прямому эфиру (`Webinar-Prep`).

## Что НЕ меняем (гарантии качества)

Эти правила — причина, по которой ТЗ работает:

- **Монтажёр ничего не ищет в записи.** ТЗ даёт полный список таймкодов с действиями — монтажёр исполняет.
- **Внутри сканируемых окон density прежняя** (кадры 30 сек + OCR 5 сек). Сужаем зону плотного скана, а не плотность.
- **Честные границы.** whisper НЕ размечает кашли/«эээ»/затупы (это аудио) — выносить отдельным правилом монтажёру; даты на экране ловит OCR; границы реза — по пословным таймкодам whisper `.json`.
- **Даты на слайдах-нарративе не трогаем** — чистим только даты в интерфейсах при демо и неактуальные цифры.
- **Решение «трогать/не трогать» — за человеком.** Скрипт только размечает кандидатов. Напр. даты/цены живого оффера могут не требовать монтажа сейчас, но потребовать позже — размечаем в любом случае (задел).
- **Сверка с автором перед отдачей монтажёру.** Во внешние каналы система сама не отправляет.

## Два слоя анализа

- **Слой 1 — транскрипт по ВСЕЙ записи** (whisper по аудио): даты, цены/офферы, тех. заминки, относительное время в РЕЧИ. Охват полный — устаревшая фраза может быть в любом месте.
- **Слой 2 — плотный OCR только по ОКНАМ шеринга экрана** (терминал, браузер, дашборд, таблицы, слайды с цифрами): даты и секреты на ЭКРАНЕ. Дату на экране нельзя поймать в разговорной части — там нет картинки с экрана, поэтому плотно сканируем не всю запись, а окна показа экрана.

## Алгоритм работы

### Шаг 1. Снять вводные
- Источник записи: Kinescope `video_id` или путь к файлу.
- **Окна шеринга экрана** — у автора эфира (он знает сценарий): «экран показывали ~20–25 мин, 44–46 мин, 1ч22м–1ч24м…». Это главный рычаг скорости.
- Актуальность оффера/цен — если в записи живой оффер, уточнить у автора, режем или это задел.

### Шаг 2. Запустить драйвер-скрипт
Операционный пайплайн (скачать → аудио → whisper → кадры → OCR → сборка) тянет **главный оркестратор** (субагент длинный Bash/мониторинг не тянет). См. блок «Драйвер-скрипт» ниже.

### Шаг 3. Прогнать черновик через `video-edit-assistant`
Скрипт выдаёт черновик-кандидаты. `video-edit-assistant` квалифицирует находки (интерфейс vs слайд-нарратив), отсеивает OCR-шум и широкие срабатывания, принимает решения «резать/блюрить/оставить», доводит до формата шаблона.

### Шаг 4. Сверить с автором и оформить финал
Показать автору, внести правки, сохранить финал в `reports/[YYYY-MM]_[тема]_montazh-tz.md`. Только потом автор передаёт монтажёру.

## Драйвер-скрипт `montage_tz.py`

```
python3 skills/Montage-TZ/montage_tz.py \
    --source <kinescope_id|путь_к_видео> \
    --windows "20m-50m,1h15m-1h40m" \
    --theme webinar-tema
```

**Ключевые опции (безопасные дефолты под слабые машины):**
- `--whisper-model base` — быстро; `small`/`medium` на мощных машинах для качества.
- `--quality 480p` — читаемо для экранного текста, легче полного файла.
- `--ocr-lang rus+eng` (дефолт) — ловит и русские даты на экране, и латиницу/терминал. Для чисто-английского UI можно `eng` (легче). Нужен установленный `rus.traineddata` (`brew install tesseract-lang`).
- `--ocr-step 5`, `--frame-step 30` — плотность скана внутри окон.
- `--windows` — окна шеринга экрана: `20m-50m,1h15m-1h40m` или `MM:SS`/`HH:MM:SS`. Без них экранный слой пуст (только речь).
- `--refresh` — пересобрать, игнорируя кэш.

**Что на выходе:** `reports/[YYYY-MM]_[тема]_montazh-tz-draft.md` — черновик по разделам (заблюрить/заменить на экране · вырезать из речи · даты в речи · чистка речи · карта структуры) с пометкой «ЧЕРНОВИК, решает человек».

**Кэш** по записи (транскрипт, кадры) — повторный заход не переделывает. Артефакты — в `source/.montage-cache/` (gitignore).

## Известные ограничения (v1)

- Дата словами («третьего июня») не ловится — паттерн ждёт цифры; в речи рядом обычно есть «сегодня/в N году», которые ловятся.
- OCR даёт шум (битые годы/токены, мисриды) — это **черновик**, `video-edit-assistant` и человек отсеивают.
- Речевой слой высокочувствительный (лучше перебдеть в черновике).
- Авто-засечки окон шеринга нет — окна задаёт автор.
- OCR русского интерфейса требует доустановки `rus`.

---

## Опорные правила

- Скилл собирает **кандидатов**, не финал. Финал — после `video-edit-assistant` и сверки с автором.
- Полнота важнее краткости: лучше лишний кандидат, чем пропущенная дата на экране.
- Не отдавать ТЗ во внешние каналы самостоятельно.
- Окна шеринга — главный рычаг скорости: всегда пытаться получить их у автора до прогона.
