# Image-Prompt

Перевод задачи на визуал в готовый промпт, который можно скопировать в любой генератор изображений.

---

## Когда использовать скилл

Активировать, когда нужен визуал для:
- Афиши вебинара / акции / интенсива
- Карточек для тг-канала и бота (анонс, напоминание, последний шанс)
- Превью и баннеров для лендинга
- Обложки лид-магнита
- Иллюстраций к прогревочной серии постов
- Иконок для смысловых блоков лендинга
- Превью статьи блога

**Не использовать для:** генерации самого изображения (этим занимается отдельный сервис), правок готовых картинок, дизайна в Figma / Photoshop.

---

## Алгоритм работы

### Шаг 1. Собрать вводные

Что обязательно вытащить из задачи:
- **Назначение визуала:** афиша, карточка, превью, иллюстрация, иконка
- **Площадка:** где будет жить (тг-канал, бот, лендинг, почта, VK, Instagram)
- **Соотношение сторон и формат:** квадрат 1:1, вертикаль 9:16, горизонталь 16:9, обложка email и т. д.
- **Будет ли текст на изображении:** да / нет / только смысловое пятно для наложения текста сверху
- **Тема и сюжет:** что должно быть на картинке (человек за ноутбуком, абстракция, объект, сцена)
- **Настроение:** деловое, энергичное, спокойное, технологичное, тёплое
- **Стиль:** фото, иллюстрация, 3D, минимализм, ретро, киберпанк и т. п.
- **Цветовая палитра:** если есть привязка к лендингу или бренду — указать
- **Что точно НЕ должно быть:** негативные элементы (логотипы конкурентов, лица знаменитостей, текст внутри картинки, стоковые позы)

Если каких-то вводных нет — задать 1–2 уточняющих вопроса. Не додумывать стиль и цвет за Ольгу.

### Шаг 2. Определить формат под генератор

По умолчанию собираем **универсальный промпт на естественном языке** (русский или английский — уточнить у Ольги, какой используется в её инструменте).

Если задача предполагает конкретный генератор — адаптировать под его особенности:
- **Midjourney:** короткие фразы через запятую, технические параметры в конце (`--ar 16:9 --style raw --s 250 --v 6`)
- **DALL-E / ChatGPT:** развёрнутые предложения, описательный язык, без технических флагов
- **Flux / Stable Diffusion:** ключевые слова с весами `(word:1.3)`, отдельный negative prompt, при необходимости LoRA-теги
- **Sora / видео-генераторы:** добавить описание движения камеры и динамики

### Шаг 3. Собрать промпт по структуре

Универсальный каркас (по этому порядку):

1. **Субъект** — кто или что в центре кадра
2. **Действие / сцена** — что происходит, в каком окружении
3. **Стиль** — фотореализм, иллюстрация, 3D-рендер, минимализм и т. д.
4. **Композиция** — план (крупный, средний, общий), точка съёмки, расположение объектов
5. **Свет и атмосфера** — естественный / студийный / контровый, тёплый / холодный, мягкий / резкий
6. **Цветовая палитра** — доминирующие цвета, акценты
7. **Технические параметры** — соотношение сторон, разрешение, флаги генератора
8. **Negative prompt** — что не должно появиться (если генератор поддерживает)

### Шаг 4. Оформить файл

Складывать в директорию проекта по правилу именования из CLAUDE.md:

`[дата]_[тема]_visual-brief.md`

Один файл = один запуск, все визуалы внутри блоками. Если визуалов много — подписать каждый блок (афиша / карточка-напоминание / обложка лид-магнита).

Внутри блока:
- Назначение визуала и канал
- Формат и соотношение сторон
- Промпт (готовый к копированию)
- Negative prompt (если есть)
- Комментарий: что должно получиться, на что обратить внимание

### Шаг 5. Передать Ольге на проверку

Промпт — это черновик. Никакой генерации от имени Ольги, никаких загрузок во внешние сервисы без подтверждения. Готовый файл уходит на проверку, после правок — обновить.

## Структура промпта в деталях

### Субъект
Кто или что в центре. Конкретика важнее обобщений.
- Плохо: «человек»
- Хорошо: «женщина 30 лет, спокойное лицо, в светлой блузке, смотрит на экран ноутбука»

### Сцена и окружение
Где это происходит, что вокруг.
- «Светлый минималистичный офис, большое окно, на столе ноутбук и керамическая кружка, утренний свет»

### Стиль
Один-два стилевых якоря, не пять разных.
- «Современная редакционная фотография, чистый журнальный стиль»
- «Плоская векторная иллюстрация в стиле Notion, без градиентов»
- «3D-рендер в стиле Apple, мягкий softbox-свет, нейтральный фон»

### Композиция
- План: extreme close-up / close-up / medium / wide / extreme wide
- Точка съёмки: уровень глаз / низкая / верхняя / диагональная
- Правило третей, центральная композиция, симметрия

### Свет
- Источник: естественный / студийный / контровый / неоновый
- Температура: тёплый / нейтральный / холодный
- Жёсткость: мягкий рассеянный / резкий с тенями
- Время суток (для естественного света): золотой час / полдень / синий час

### Палитра
- 2–4 доминирующих цвета
- Акцентный цвет, если нужен
- Привязка к фирстилю Zerocoder — если есть, указать в задаче

### Технические параметры
- Соотношение сторон под площадку (см. таблицу ниже)
- Качество / разрешение
- Флаги генератора (только если знаем точный инструмент)

### Negative prompt
- Текст на изображении (если он мешает или будет наложен в дизайне)
- Лишние руки, искажённые пальцы, странные лица
- Логотипы, водяные знаки
- Стоковые штампы: рукопожатие, лампочка-идея, человек у графиков с указкой
- Низкое качество, размытость, артефакты

## Форматы под каналы

| Канал | Соотношение | Особенности |
|---|---|---|
| Афиша вебинара (универсал) | 1:1 (1080×1080) | Под почту, ленту, превью карточки |
| TG-канал / бот, превью поста | 1.91:1 (1200×630) | Свернётся в широкий баннер |
| TG-канал, карточка для ленты | 4:5 (1080×1350) | Вертикаль под просмотр в ленте |
| Stories / Reels / TG-история | 9:16 (1080×1920) | Вертикаль на весь экран |
| Лендинг, первый экран | 16:9 (1920×1080) или шире | Часто с местом под текст |
| Лендинг, превью блока | 4:3 или 1:1 | Внутри сетки блоков |
| Обложка email | 2:1 или 3:1 | Узкая горизонталь под шапку письма |
| Обложка лид-магнита | 3:4 или 4:5 | Под превью PDF-материала |

## Шаблоны под типовые задачи

### Афиша вебинара (универсальная)

```
Современная редакционная фотография на тему [тема вебинара].
В центре кадра — [описание субъекта, например: ноутбук на светлом столе, на экране абстрактная визуализация данных].
Окружение: [описание сцены]. Композиция: средний план, уровень глаз, правило третей.
Свет: мягкий естественный из окна слева, утренний золотой час.
Палитра: нейтральные тёплые тона, акцент — [акцентный цвет].
Чистая композиция с зоной слева для наложения заголовка.
Соотношение 1:1, фотореализм, высокая детализация.

Negative: текст на изображении, водяные знаки, стоковая постановка, искажённые руки, размытость.
```

### Карточка-напоминание в бот

```
Минималистичная векторная иллюстрация: [объект, символизирующий тему — например: цифровой будильник с подсветкой].
Плоский стиль, чистые формы, без градиентов и теней.
Палитра: 2–3 цвета, основной [цвет], акцент [цвет], фон светлый нейтральный.
Композиция центрированная, много воздуха вокруг объекта.
Соотношение 1.91:1, под превью поста в Telegram.

Negative: текст, реализм, фотография, лишние детали.
```

### Обложка лид-магнита (PDF-материал)

```
Обложка цифрового документа на тему [тема ЛМ].
В центре — [объект-метафора темы, например: открытая книга с световыми частицами над страницами].
Стиль: современный редизайн обложки бизнес-книги, чистая типографическая сетка, премиальное ощущение.
Палитра: глубокий [основной цвет] фон, акцентный [цвет] для деталей.
Композиция вертикальная, верхняя треть свободна под заголовок, нижняя — под имя автора и логотип.
Соотношение 3:4, разрешение под печать и веб.

Negative: реалистичная книга на столе, стоковая постановка, размытость, лишний текст.
```

### Иллюстрация к прогревочному посту

```
Концептуальная иллюстрация на тему [месседж поста, например: разница между ручной работой и автоматизацией].
[Описание сцены, например: две комнаты в одном кадре — слева хаос из бумаг и часов, справа чистое пространство с одним светящимся экраном].
Стиль: редакционная иллюстрация в духе New Yorker, мягкие линии, ограниченная палитра.
Палитра: приглушённые тона, акцент — тёплый [цвет] на правой части кадра.
Композиция: симметричное разделение по центру, средний план.
Соотношение 4:5 под ленту.

Negative: реализм, фотография, текст, перегрузка деталями.
```

## Чек-лист перед сдачей

- [ ] Промпт даёт ровно одну картинку, а не «и так, и так»
- [ ] Стиль указан конкретно (не «красиво», не «современно»)
- [ ] Композиция учитывает место под текст, если он будет накладываться поверх
- [ ] Соотношение сторон соответствует площадке
- [ ] В negative prompt вынесено всё, что генератор любит вставлять не туда (стоковые штампы, лишний текст, искажения)
- [ ] Нет привязки к конкретному человеку (известная личность, спикер) без явного разрешения
- [ ] Файл назван по формуле из CLAUDE.md
- [ ] Если нужны разные форматы под разные площадки — собраны отдельными блоками, а не один промпт «для всего»

---

## Опорные правила

- **Не выдумывать бренд-стиль.** Если гайдлайнов Zerocoder по визуалу нет — спросить, на какой стиль ориентироваться, или предложить 2–3 варианта на выбор.
- **Конкретика важнее красоты.** «Мягкий контровой свет в полтретьего дня» работает лучше, чем «красивый свет».
- **Один промпт — одна задача.** Не пытаться втиснуть афишу, карточку и обложку в одну формулировку.
- **Promt — это черновик.** Готовое изображение сначала показывается Ольге, никакой автоматической публикации.
- **Парность с AI-Copywriter.** Если задача — анонс под канал, тексты собирает AI-Copywriter, визуал — этот скилл. В файл `[дата]_[тема]_visual-brief.md` ссылка на парный `_announce.md`.
