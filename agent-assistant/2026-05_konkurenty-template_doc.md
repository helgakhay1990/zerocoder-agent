# Шаблон сбора конкурентов

Инструкция для агента: базовый каркас таблицы конкурентов. Для каждой новой задачи **пул конкурентов формируется заново под нишу и тематику** — постоянного списка нет. Базовые поля сохраняются, набор конкурентов и доп.поля — меняются под запрос.

---

## Принципы

1. **Пул конкурентов — под задачу.** В разных запусках смотрим разных игроков: для Python — одни, для 1С — другие, для детских курсов — третьи. Не переиспользуем старый список без проверки.
2. **Источник всегда фиксируется.** Ссылка на сайт/посадку — обязательное поле, без неё запись не валидна.
3. **Никаких догадок.** Цены, форматы, состав программы — только из явно указанной информации на сайте/в лендинге/в воронке. Если данных нет — пишем «нет данных», а не предполагаем.
4. **Дата сбора** ставится рядом с каждой записью — рынок меняется быстро.

---

## Базовый каркас (обязательные поля для любой ниши)

| Поле | Что заполняем |
|---|---|
| **Конкурент** | Название компании / школы / автора |
| **Ссылка** | URL продукта или посадочной |
| **Позиционирование** | Одной строкой — что и кому продают |
| **Формат продукта** | Курс / интенсив / марафон / вебинар / мини-курс / воронка-бесплатник / клуб |
| **Длительность** | Сколько идёт (недели/месяцы) |
| **Цена** | Цифра + валюта. Бесплатник — пометить «бесплатно». Если несколько тарифов — все |
| **Условия оплаты** | Рассрочка, скидки, гарантия возврата (если упомянуто) |
| **Состав программы** | Модули, инструменты, ключевые темы — тезисно |
| **Бесплатник / ЛМ** | Есть ли вход через бесплатное (вебинар, мини-курс, гайд, бот). Какой формат и тематика |
| **Канал продажи** | Как затягивают: вебинар → продажа, бот → воронка, e-mail-цепочка, диагностика |
| **Фишки / отличия** | Что цепляет, нестандартные механики, сильные обещания |
| **Источник данных** | Скрин или ссылка на конкретную страницу, где взяли инфу |
| **Дата сбора** | YYYY-MM-DD |

---

## Доп.поля под нишу (включаются в зависимости от задачи)

Подбираются исходя из специфики запуска. Примеры:

- **Детские курсы:** возраст ребёнка, формат занятий (онлайн/гибрид), сертификат, родительский контроль.
- **Курсы для пенсионеров:** уровень сложности подачи, поддержка/наставник, оффлайн-встречи.
- **Курсы для госслужащих:** соответствие нормативам, удостоверение, корпоративные тарифы.
- **Программирование (Python / JS / 1С / мобилка):** язык/стек, итоговый проект, помощь с трудоустройством, упор на ИИ-инструменты.
- **Промт-инженер / ИИ-студии:** какие нейронки изучают, есть ли практика по разработке студии.
- **Чат-боты / Сейлбот:** платформа (Сейлбот, BotHelp, Telegram-боты), уровень (новичок/продвинутый).
- **Веб-дизайн / no-code:** инструменты (Тильда, Фигма, Креатиум, WordPress), упор на ИИ.
- **Карьерные консультации / диагностики:** как приглашают, бесплатна ли первая консультация, что предлагают на выходе.

Список не закрытый — под каждую задачу формируется свой набор.

---

## Формат итогового артефакта

- **По умолчанию** — Google-таблица. Один лист = один формат продукта (курс / вебинар / интенсив и т. д.), если в задаче нужно разделить.
- **Если в задаче явно сказано** «таблица по продуктам с листом на каждый продукт» — каждый продукт = свой лист.
- **Платные и бесплатные продукты** — на отдельных листах, если в задаче это разделение указано.
- В первой строке — заголовки, цвета/форматирование — по аналогии с предыдущими таблицами Ольги.

---

## Алгоритм работы агента

1. Прочитать задачу: ниша, формат (курсы / вебинары / бесплатники / всё вместе), сколько конкурентов нужно (обычно 5–7 по каждому формату).
2. Уточнить у Ольги, если непонятно: какие именно форматы интересуют, есть ли референс прошлой похожей таблицы.
3. Подобрать пул конкурентов **под эту конкретную задачу** — не использовать старый список «по памяти».
4. По каждому конкуренту собрать базовый каркас + релевантные доп.поля под нишу.
5. Зафиксировать источник для каждой записи.
6. Сделать короткое саммари в начале: ключевые наблюдения (что повторяется у всех, что выделяется, есть ли тренды по форматам/ценам).
7. Отдать ссылку на таблицу + саммари в чат.

---

## Чего не делаем

- Не придумываем цены и состав программ, если на сайте этого нет — пишем «нет данных».
- Не берём в выборку крупных игроков (Нетология, Скиллбокс, Яндекс.Практикум), если в задаче явно сказано «небольшие/локализованные/нишевые школы».
- Не копируем тексты конкурентов дословно — только тезисное изложение.
