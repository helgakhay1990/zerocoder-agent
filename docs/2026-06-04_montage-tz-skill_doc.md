# Проект: скилл «Montage-TZ» — упрощение монтажной ветки и шеринг с коллегами

**Дата:** 2026-06-04 · **Статус:** дизайн на согласовании (этап 1 из 3)
**Зачем:** сборка ТЗ на монтаж занимает почти рабочий день на запись. Цель — сократить время и упаковать пайплайн в переносимый скилл, который коллеги запускают сами **на любой записи эфира**.

> Это **общий** инструмент под любой эфир, не под конкретную запись. Опыт собран на двух реальных прогонах — Пименов (22.05.2026) и OpenClaw (03.06.2026); они служат источником уроков, а не предметом документа.

---

## Полный объём ТЗ на монтаж (что скилл должен покрывать)

Скилл готовит ТЗ по всем категориям монтажа, не только по блюру. Из чек-листа `agent-assistant/2026-05_montazh-checklist_guide.md`:

| Категория | Что ищем | Чем ловится |
|---|---|---|
| **Вырезать** | временные маркеры в речи, старые цены/офферы, речевой мусор, тех. заминки | транскрипт (речь) + слух (мусор) |
| **Заменить слайд/кадр** | устаревшие даты на экране, опечатки, неактуальные цены на слайдах | визуальный проход |
| **Заблюрить** | личные данные, даты/время в интерфейсах, чужие переписки, внутренние ссылки, уведомления | визуальный проход + OCR |
| **Чистка речи** | «эээ», «ну», повторы, длинные паузы | слух (whisper это НЕ размечает) |

Доля категорий у каждого эфира разная: где-то главное — блюр секретов в демке, где-то — вырезать устаревший оффер в речи, где-то — заменить слайды с ценами. Скилл одинаково готов ко всем.

---

## Два слоя анализа (важное разделение)

Упрощение строится на том, что разные категории ловятся разными проходами с разным охватом:

- **Слой 1 — транскрипт по ВСЕЙ записи.** whisper по аудио целиком. Даёт всё, что в речи: произнесённые даты, цены, имена, тех. заминки, повторы. Охват полный, потому что устаревшая фраза может быть в любом месте.
- **Слой 2 — плотный визуальный скан ТОЛЬКО по экранным окнам.** Кадры + OCR там, где идёт демонстрация экрана (терминал, браузер, дашборд, слайды с цифрами). Дату/секрет на экране нельзя поймать в разговорной части — там нет картинки с экрана. Поэтому плотно сканируем не всю запись, а окна шеринга экрана.

Гарантия качества от этого не страдает: внутри визуальных окон density прежняя (30 сек кадры + 5 сек OCR), мы лишь не молотим OCR-ом разговорные блоки, где экранных дат физически нет.

---

## Что НЕ меняем (гарантии качества)

Эти правила — причина, по которой ТЗ работает. Любое упрощение их сохраняет:

- **Монтажёр ничего не ищет в записи.** ТЗ даёт полный список таймкодов с действиями, монтажёр исполняет.
- **Внутри сканируемых окон density прежняя** — сужаем зону плотного скана, а не его плотность.
- **Честные границы.** whisper не размечает кашли/«эээ» (это аудио); даты на экране ловит OCR/visual-scanner; границы реза — по пословным таймкодам whisper `.json`.
- **Даты на слайдах-нарративе не трогаем** — чистим только даты в интерфейсах при демо и неактуальные цифры.
- **Решение «трогать/не трогать» — за автором.** Напр., даты и цены живого оффера могут НЕ требовать монтажа сейчас, но потребовать позже, когда эфир останется в ротации, а оффер протухнет. Скилл их размечает в любом случае (задел), но применять правки — по решению автора.
- **Сверка с автором** перед отдачей монтажёру; во внешние каналы система сама не отправляет.

## Где уходит время (общие узкие места)

| Узкое место | Почему долго | Рычаг |
|---|---|---|
| Тяжёлая модель whisper (`medium`+) на длинной записи | медленно, душит слабые машины | `base`/`small` по умолчанию |
| OCR/плотные кадры по ВСЕЙ записи | большинство кадров — разговор без экранных дат | плотный скан только по окнам шеринга |
| Скачивание целого файла записи | гигабайты трафика и времени | 480p, по возможности только нужные сегменты |
| Ручной запуск каждого этапа | накладные на оркестровку | один драйвер-скрипт |

---

## Пять рычагов упрощения

1. **Плотный визуальный скан только по окнам шеринга экрана** (см. «Два слоя»). Снимает основной пустой молот.
2. **whisper мельче и по аудио.** `base`/`small` хватает для поиска дат/цен/заминок в речи; безопасный дефолт `base`. Транскрибируем извлечённое аудио, не видео.
3. **480p, в идеале только нужные сегменты.** Читаемо для экранного текста, вдвое легче полного файла.
4. **Вход от автора на старте — где были окна шеринга экрана.** Автор обычно знает сценарий: «экран показывали ~0:20–0:50 и 1:15–1:40». Это мгновенно сужает зону визуального скана без авто-разведки.
5. **Кэш артефактов** по ID записи: кадры, транскрипт, OCR не переделываются при повторном заходе.

## Архитектура скилла

Сейчас знание размазано: агент `video-edit-assistant` + подмастерья `transcript-analyzer`/`visual-scanner` + чек-лист монтажа. Скилл собирает операционный слой в один переносимый пакет.

```
skills/Montage-TZ/
  SKILL.md              # чек-лист-инструкция: шаги, границы, гарантии
  montage_tz.py         # драйвер-скрипт (ядро упрощения)
  templates/
    montage-tz-template.md   # скелет ТЗ (берём существующий шаблон)
  README-coleagues.md   # как запустить с нуля + что поставить
```

**Разделение зон (не дублируем агентов):**
- **Скилл = операционный слой** (скачать → аудио → whisper → окна шеринга → точечный визуальный скан/OCR → собрать черновик). Тянет главный оркестратор (субагент длинный Bash/мониторинг не тянет — проверено).
- **Агенты = суждение.** `visual-scanner` квалифицирует находки (интерфейс vs слайд-нарратив), `video-edit-assistant` принимает решения «вырезать/заменить/блюр/оставить» по всем категориям и финально собирает ТЗ. Скилл их кормит сырьём, не заменяет.

## Драйвер-скрипт `montage_tz.py` (этап 2)

**Вход:**
- источник: Kinescope `video_id` (через MCP/API) ИЛИ путь к локальному файлу;
- окна шеринга экрана: список интервалов (от автора; если пусто — авто-засечка по транскрипту/смене сцены);
- опции с безопасными дефолтами: `--whisper-model base`, `--quality 480p`, `--ocr-step 5`, `--frame-step 30`.

**Шаги:**
1. Получить запись (480p; по возможности — только сегменты окон).
2. Извлечь аудио → whisper (`base`) → транскрипт `.json/.srt/.txt` (кэш по ID). **Слой 1 по всей записи.**
3. По транскрипту — кандидаты речевых категорий: даты/цены/имена/заминки/повторы с таймкодами.
4. Карта 10 мин (мозаика) — общая структура; если окна не заданы — предложить их отсюда + транскрипт.
5. **Слой 2:** по окнам шеринга — кадры 30 сек + OCR 5 сек (`tesseract`) → перечень таймкодов с экранными датами/секретами.
6. Собрать **черновик ТЗ** из шаблона по всем категориям: «Вырезать» (из транскрипта), «Заменить слайд/Заблюрить» (из визуального слоя), «Чистка речи» (отдельным правилом — whisper не размечает), честные пометки «не отсмотрено» где визуала не было.

**Выход:** `[дата]_[тема]_montazh-tz.md` — черновик на проверку `video-edit-assistant` → автору. Многошаговый ручной вечер → «дай ID + окна → черновик за минуты».

## Универсальность для коллег (среда пока неизвестна)

- **Безопасные дефолты:** whisper `base`, 480p — работают и на слабых машинах. Мощным можно поднять флагами.
- **Зависимости** в `README-coleagues.md`: `ffmpeg`, `tesseract` (+ rus/eng), `whisper.cpp`/`faster-whisper`, python-пакеты. Предполётная проверка наличия с понятным сообщением.
- **Kinescope-токен** у каждого свой (через `.mcp.json`) — в скрипт не зашиваем.
- Уточнить реальную среду коллег перед финальной упаковкой — подстроить дефолты.

## Открытые вопросы

- Среда коллег (ОС, мощность, что уже стоит) — влияет на дефолты и инструкцию установки.
- Авто-засечка окон шеринга по транскрипту/смене сцены — насколько точна без подсказки автора (проверить на этапе 2).
- Тянуть по HLS только сегменты окон или проще весь 480p (зависит от длины записей и трафика).
