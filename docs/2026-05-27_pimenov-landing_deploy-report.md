## 4. План реального деплоя на Beget

Если бы проект публиковался на Beget по схеме урока, шаги выглядели бы так:

### 4.1. Регистрация
1. Открыть [beget.com](https://beget.com) → **Регистрация**.
2. Заполнить почту, телефон, пароль; подтвердить почту.
3. На тарифной странице выбрать «Blog» (~219 ₽/мес) — для статического HTML минимального тарифа достаточно. У Beget есть 30 дней бесплатного теста — на сдачу ДЗ хватит.

### 4.2. Создание сайта в панели
1. Войти в личный кабинет → раздел **Сайты**.
2. **Добавить новый сайт** → задать имя, например `pimenov-landing`.
3. Beget автоматически создаст папку `pimenov-landing/public_html/` — это и есть корневая директория, откуда хостинг отдаёт файлы при запросе сайта в браузере.

### 4.3. Загрузка архива
1. В панели открыть **Файловый менеджер** → перейти в `pimenov-landing/public_html/`.
2. Удалить дефолтные файлы Beget (`index.html`-заглушка, иногда `.htaccess`, служебные).
3. Кнопка **Загрузить файлы** → выбрать `2026-05_webinar-pimenov_landing.zip`.
4. Правый клик по архиву → **Распаковать**. Файлы лягут прямо в `public_html` благодаря структуре архива без вложенной папки.
5. После распаковки сам архив можно удалить, чтобы не висел лишним.

Внутри `public_html/` после распаковки:
```
index.html
styles.css
script.js
images/
```

### 4.4. Проверка по техническому домену
Beget автоматически выдаёт каждому сайту бесплатный технический домен вида `pimenov-landing.beget.tech`. Открываешь его в браузере — лендинг должен загрузиться полностью со шрифтом, картинками и якорями.

### 4.5. Привязка собственного домена (опционально)
Если есть свой домен у стороннего регистратора:
1. В Beget → **Домены** → **Добавить домен**.
2. В личном кабинете регистратора заменить NS-серверы на `ns1.beget.com` / `ns2.beget.com`.
3. Дождаться обновления DNS (от 1 до 24 часов).
4. В Beget привязать домен к сайту: **Сайты → твой сайт → Привязка домена**.

### 4.6. SSL через Let's Encrypt
1. В панели → раздел **SSL-сертификаты** (или внутри карточки сайта).
2. Выбрать домен → **Заказать Let's Encrypt** (бесплатно).
3. Дождаться статуса **Активен** (1–10 минут).
4. Включить опцию **Перенаправление HTTP → HTTPS**.
5. Открыть сайт по `https://...` — в адресной строке должен появиться замочек.

### 4.7. Что такое `public_html` и зачем
`public_html` — это **корневая публичная папка сайта** на классических Linux-хостингах. Apache или nginx настроены так, что при запросе домена сервер отдаёт файлы именно из неё. Всё, что лежит **выше** `public_html` (в `~/`, `~/logs/`, `~/private/` и т.п.), наружу не выдаётся — там хранятся логи, конфиги, файлы вне публичного доступа. Файл `index.html` в корне `public_html` отдаётся, когда пользователь открывает голый домен без указания файла.

## 5. Бонус — фактическая публикация в интернете

Параллельно с описанием логики Beget проект опубликован через **Netlify Drop** — бесплатный сервис для статики с автоматическим SSL и глобальным CDN, без регистрации и без CLI.

### 5.1. Почему не Vercel
Первой попыткой был Vercel CLI. Установила глобально (`npm install -g vercel`), запустила `vercel login` через device-flow. Vercel показал ссылку с кодом, перешла в браузере — Vercel потребовал верификацию по SMS. На российские номера SMS не доходит (известная проблема с их провайдером), на иностранные номера — после нескольких попыток — сработал rate-limit «Too many phone numbers have been attempted for this account». Вход в Vercel оказался заблокирован, переключилась на Netlify.

### 5.2. Netlify Drop
1. Открыла [app.netlify.com/drop](https://app.netlify.com/drop).
2. Перетащила ZIP-архив `2026-05_webinar-pimenov_landing.zip` прямо в зону загрузки на странице.
3. Netlify за 20 секунд распаковал архив, развернул сайт на CDN и выпустил SSL-сертификат.
4. Получила автоматическую публичную ссылку вида `grand-nasturtium-641f41.netlify.app`.
5. В настройках сайта переименовала поддомен на осмысленный — `pimenov-pshinnik-web` (по фамилиям спикеров эфира). Старый URL автоматически редиректит на новый.

### 5.3. Результат

**🌐 Сайт в интернете:** https://pimenov-pshinnik-web.netlify.app

Проверка через `curl -I`:
```
HTTP/2 200
content-type: text/html; charset=UTF-8
cache-status: "Netlify Edge"
etag: "e9d66b52bff50693b03f0924a9925898-ssl"
```

- **HTTP 200** — сайт отвечает корректно.
- **HTTPS активен** — соединение по HTTP/2, что работает только поверх TLS, значит SSL установлен. Netlify выпустил Let's Encrypt-сертификат автоматически в момент деплоя.
- **CDN работает** — ответ идёт с Netlify Edge, заголовок `etag` с суффиксом `-ssl` подтверждает раздачу зашифрованного контента.
- **Сайт открывается визуально** — заголовок страницы «Бизнес-разбор с ИИ‑консультантом Сергеем Пименовым — 22 мая, 13:00 МСК», все блоки на месте, картинки спикеров и шрифт Inter грузятся (проверено через headless Chrome).

## 6. Комментарий: что получилось, какие сложности были

**Получилось:**
- лендинг приведён в чистый вид: только файлы сайта в корне, главный файл — `index.html`;
- ZIP собран без лишнего, готов к загрузке на любой хостинг с файловым менеджером;
- проект работает локально, все ключевые элементы (шрифты, картинки, кнопки, якоря) проверены;
- описаны шаги полного цикла деплоя на Beget по логике урока;
- сверху — реальная публикация на Vercel для живой ссылки.

**Сложности:**
- В исходной папке `homework-card/` был файл-обёртка `index.html`, который через `<meta refresh>` редиректил на `2026-05_webinar-pimenov_landing.html`. Это рабочий вариант, но он добавляет лишний HTTP-хоп. Решение: переименовала основной файл лендинга в `index.html`, а старый редирект удалила. Теперь корневой домен сразу открывает лендинг без перенаправлений.
- Файл ТЗ `.docx` лежал рядом с файлами сайта. Если упаковать «всё подряд», ТЗ попадёт на хостинг. Решение: ZIP собран явным списком файлов (`index.html`, `styles.css`, `script.js`, `images/`), без `*.docx` и скрытых файлов.
- Локальная проверка через `file://` не подходит для лендинга со ссылками на внешние шрифты и относительными путями — Chrome и Safari могут блокировать загрузку из-за CORS. Решение: запущен локальный HTTP-сервер на 8765 порту, лендинг открыт по `http://localhost:8765/`.

**Итог:** проект готов к публикации, понимание процесса деплоя на классический хостинг (Beget) и современный JAMstack-сервис (Vercel) сформировано.
