## Маршрут проекта

### Для кого

- **Прямой пользователь — я.** Система снимает с меня рутину запусков: черновики анонсов, структуры лендингов, концепции, гайды, ТЗ на монтаж — на проверку, а не с нуля.
- **Косвенно — команда и спикеры:** быстрее готовые материалы, единый тон, меньше фактических ошибок (всё из knowledge-base, а не из головы).

### Какой сценарий должен работать

Эталонный сквозной сценарий «Запуск вебинара» без ручного склеивания:

1. `launch-strategist` собирает концепцию (внутри — `audience-researcher` + `offer-mechanic`).
2. `landing-architect` разворачивает концепцию в структуру лендинга.
3. `announce-writer` готовит анонсы по волнам (email × 2, tg-канал, tg-бот).
4. `methodist` собирает презентацию эфира и гайд участника.
5. **`Webinar-Prep`** (новый скилл) собирает операционную готовность к эфиру.
6. После эфира — `video-edit-assistant` готовит ТЗ на монтаж.
7. `editor` проходит финалом по всем текстам — с учётом регистра бренда (Zerocoder / IU).

Критерий «работает»: я запускаю цепочку, получаю пакет черновиков, правлю точечно — а не переписываю.

### Как проверю результат

- **Боевой прогон:** запустить оркестр на реальном ближайшем вебинаре и замерить, сколько правок руками понадобилось на каждом этапе.
- **Метрика подпорок:** считаю, сколько раз пришлось вмешаться вручную помимо финальной вычитки. Цель к концу месяца — свести к точечным правкам.
- **Фактчек:** ни одного выдуманного факта/цены/регалии в выходных материалах (всё трассируется к knowledge-base).
- **Тон по бренду:** материалы Zerocoder — живой регистр, материалы IU — академический; `editor` различает их сам.

### Как презентую

- **Демо-прогон** оркестра на одном вебинаре: показываю вход (бриф) → выход (пакет черновиков) и лог, кто из субагентов что сделал.
- **Документ-карта системы:** схема уровней + таблица «задача → субагент → источник правды».
- Этот файл плана + подтверждающие скриншоты доработок как точка отсчёта.

**Что сделала** (файл `.claude/agents/editor.md`):
- Добавила в алгоритм отдельный шаг — *«Определи регистр по проекту/бренду»* — перед вычиткой.
- Добавила раздел **«Регистр по проекту»** с таблицей-сравнением Zerocoder / IU: что НЕ считать ошибкой и что всё равно режем в каждом регистре.

- Добавила принцип «регистр по бренду решает раньше, чем сегмент».
