## №2 (СЛЕДУЮЩИЙ). ТЗ на монтаж через Telegram-бот

**Идея:** Повелительница кидает в бот ссылку на запись активности + описывает явно известные моменты («спикер назвал цену X», «проговорил дату Y») → агент сам проверяет остальное по нашим классическим нюансам монтажа и возвращает черновик ТЗ.

**Почему быстро:** ядро уже есть — скилл `Montage-TZ` (`montage_tz.py`: транскрипт по всей записи + OCR по окнам экрана → черновик ТЗ) + бот `launch-pult`. Это интеграция + UX, не новая способность.

**Чего не хватает (объём работ):**
- Сценарий «долгой задачи» в боте: сейчас бот отвечает быстрыми текстами; монтаж — минуты-десятки минут. Нужен async-флоу: принял ссылку+заметки → запустил пайплайн в фоне → прислал готовый черновик/ссылку в чат.
- Кнопка/команда «🎬 ТЗ на монтаж» → бот спрашивает ссылку + (опц.) заметки и окна показа экрана.
- Известные моменты пользователя → затравка: фиксируются как высокоуверенные строки, агент дополняет остальным из транскрипта/OCR.

**Оговорки:**
- Пайплайн крутится на Mac (где бот): whisper на 2,4 ч ~10-20 мин фоном, бот пингует по готовности.
- Окна показа экрана для OCR пока задаёт человек (авто-детект — отдельный шаг развития Montage-TZ). Без окон — скан всей записи (медленнее, но без участия).
- Выход — **черновик на сверку**, не финал сразу монтажёру (наше правило обязательной сверки).

**MVP:** бот-флоу «ссылка + заметки → фоновый montage_tz.py → черновик ТЗ в Telegram». Известные факты идут заметками в черновик.

## №1 (КРУПНЫЙ, позже). Разведка воронки конкурента «под ключ»

**Идея:** сущность, что регистрируется на активность конкурента и снимает инсайты по всей цепочке: лендинг регистрации → прогревочная воронка до эфира (письма, бот, напоминания) → офферы на самом эфире → пострпродажная воронка. Эту функцию (редко) выполнял живой ассистент.

**Реализуемо слоями, по сложности:**
- 🟢 Статика лендинга/оффера — уже умеем (`ai-competitors` + аналитический блок, со скрин-пруфами и уровнями уверенности).
- 🟡 Регистрация в воронку — `playwright` заполняет форму; нужен разведочный контакт (отдельный email + Telegram, иногда телефон), который Повелительница контролирует. Капчи/антибот — серая зона, агрессивно не обходим.
- 🟡 Сбор прогрева — письма в разведочный ящик (IMAP или пересылка), бот-воронка в Telegram. Многодневно, по расписанию — через облачные routines.
- 🔴 Эфир и пострпродажа — самое тяжёлое. Живой эфир в реальном времени хрупко; реалистичнее брать запись/повтор и прогонять через монтажный пайплайн (поиск офферов/цен/дедлайнов). Пострпродажа — снова сообщения во времени.

**Этика:** «тайный покупатель» — легитимная маркетинговая практика; под реальный контакт, без массовой фабрикации личностей и обхода защит.

**MVP-рецепт (80/20):** не автономная регистрация+присутствие, а разделение: человек делает редкие шаги (регистрируется разведочным email, пересылает прилетевшее, кидает ссылку на повтор), агент делает синтез — единую карту воронки (точки касания, тайминги, темы, крючки, офферы, дедлайны, пострпродажа) + инсайты «где наша возможность».

**Форма:** субагент `funnel-analyst` (структурирует цепочку) + скилл-методичка «что снимаем на каждом этапе» + трекер разведки (`_state.md` под конкурента) + routines. Опирается на готовый аналитический блок.
