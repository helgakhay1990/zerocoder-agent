# Разведчик воронки конкурента — архитектура (design-док)

**Дата:** 2026-06-05 · спроектировано с Повелительницей.
**Статус:** дизайн согласован, код не начат. Бэклог идея №1 (см. `docs/2026-06-04_backlog-idei_doc.md`).
**Связано:** скилл `ai-competitors` (статика), `Montage-TZ`/`transcript-analyzer` (эфир-запись), аналитический конвейер (уверенность/пруфы).

---

## Цель

Сущность, что проходит воронку конкурента как «тайный покупатель» и снимает инсайты по всей цепочке: лендинг регистрации → прогрев до эфира (письма, бот, напоминания) → офферы на эфире → постпродажный дожим. Раньше это редко делал живой ассистент. Итог — **единая карта воронки** + инсайты «где наша возможность для отстройки Zerocoder».

## Принцип MVP: человек собирает сырьё — агент синтезирует

Не автономная регистрация и присутствие на живом эфире (хрупко + ToS-риск), а разделение:
- **Человек** (редкие ручные шаги): регистрируется разведочным контактом, делает скрины TG-бота конкурента, кидает ссылку на повтор эфира.
- **Агент**: читает scout-почту по IMAP, превращает поток касаний в карту воронки по оси времени + инсайты.

80/20: 20% нечастого ручного труда снимают 80% хрупкости автоматизации.

---

## Слои по зрелости

| Слой | Что снимаем | Чем | В MVP |
|---|---|---|---|
| 🟢 0. Статика | лендинг регистрации, оффер, УТП, цены | `ai-competitors` | ✅ готово |
| 🟡 1. Регистрация | вход в воронку | разведочный контакт email+TG; человек (или `playwright` под надзором) | ✅ ручной шаг |
| 🟡 2. Прогрев | письма + бот-воронка во времени | **IMAP-чтение scout-ящика** (агент) + **скрины TG-бота вручную** | ✅ ядро MVP |
| 🔴 3. Эфир + постпродажа | офферы/дедлайны на эфире, дожим после | запись/повтор → `Montage-TZ`/`transcript-analyzer`; постпродажа — письма через IMAP | 🟡 частично |

Живой эфир в реальном времени в MVP не ловим — берём запись/повтор и гоним через готовый транскрипт-пайплайн (он уже ищет цены/офферы/дедлайны: `RE_PRICE`/`RE_OFFER`).

## Решения (зафиксированы 2026-06-05)

1. **Сбор прогрева:** IMAP для почты (агент читает scout-ящик), TG-бот конкурента — скринами вручную (боты трудно читать автоматом надёжно).
2. **Разведочный контакт:** email + Telegram. Телефон пока не заводим (добавим, если воронка упрётся в SMS/звонки).
3. **Подопытный №1:** прямой конкурент по вайб-кодингу/ИИ из дежурного списка `ai-competitors`. Конкретное имя — перед боевым прогоном.
4. **Периодический сбор:** без планировщика (routines/cron) в MVP. Синтез запускаем вручную, когда сырьё накопилось. Routines — задел на потом.

## Компоненты

### 1. Субагент `funnel-analyst` (новое — ядро)

Единственное по-настоящему новое: достраивает **ось времени**, которой нет в статике.

- **Вход:** разнородное сырьё — письма (выгруженные IMAP-сборщиком), скрины TG-бота, транскрипт эфира, постпродажные письма.
- **Делает:** собирает карту касаний: `день/час → канал → тема → крючок → оффер → дедлайн → эскалация дефицита`; выделяет паттерны прогрева (частота, ритм, тон), офферную логику, постпродажный дожим.
- **Выход (текст, файл пишет оркестратор):** карта воронки + блок «где наша возможность» (по образцу `ai-competitors`: «забрать механику» / «дыра для отстройки»).
- **Границы (зашить в субагента):** сам НЕ регистрируется, НЕ обходит защиты, НЕ фабрикует личности. Принимает готовое сырьё (режим как у `analyst-assistant`). Уровни уверенности 🟢/🟡/🔴 + пруфы — сквозь весь разбор.

### 2. Скилл-методичка «Разведка воронки»

Чек-лист «что снимаем на каждом слое»:
- **Регистрация:** что просят на форме, double opt-in, сразу ли welcome-письмо.
- **Прогрев:** число касаний, ритм (через сколько часов/дней), темы, типы крючков, как растёт дефицит/срочность, лид-магниты внутри.
- **Эфир:** когда называют оффер, какой, какой дедлайн, бонусы за «сейчас», цена и якоря.
- **Постпродажа:** дожим после эфира, сколько писем, падает ли цена, реактивация.
- **Этика/ToS как граница** — прямым текстом: один реальный контакт, без массовой фабрикации и обхода защит.
- Опирается на аналитический блок (уверенность/пруфы) — не дублирует его.

### 3. Трекер `_state.md` под конкретного конкурента

Лог разведки (шаблон, копируется под каждого подопытного): дата регистрации, разведочный контакт, журнал прилетевшего (дата/время → канал → что), ссылка на повтор эфира, оффер, постпродажа, текущая стадия, открытые вопросы.

### 4. IMAP-сборщик scout-почты (детерминированный скрипт)

По образцу `montage_tz.py`: маленький скрипт `scout_inbox.py` — по требованию забирает новые письма из scout-IMAP, складывает в папку как `.txt`/`.eml` для `funnel-analyst`. **Без расписания** (MVP) — запуск руками, когда нужно собрать накопившееся. Креды scout-ящика — в `.env`, не в git.

## Поток данных

```
человек: регистрация разведочным контактом email+TG (слой 1)
        │
        ▼
[прогрев во времени: письма + TG-бот]
        │                         │
   письма → scout-ящик       TG-бот → скрины вручную
        │                         │
   scout_inbox.py (IMAP, по требованию)   │
        │                         │
        └─────────► funnel-analyst ◄──────┘
                          │
   эфир: ссылка на повтор → Montage-TZ/transcript → офферы/цены/дедлайны
                          │
   постпродажа: письма (IMAP) ─────────────┐
                          ▼                 ▼
                  КАРТА ВОРОНКИ + инсайты «где наша возможность»
                  (файл пишет оркестратор, трекер _state.md обновляется)
```

## Что переиспользуем, что строим

- **Берём готовое:** `ai-competitors` (статика), `transcript-analyzer`/`Montage-TZ` (эфир-запись), аналитический конвейер (уверенность 🟢🟡🔴 + скрин-пруфы), `_state.md`-шаблон.
- **Строим новое:** `funnel-analyst` (ось времени) + методичка «Разведка воронки» + трекер-шаблон под конкурента + `scout_inbox.py` (IMAP-сборщик).

## Этика и ToS — жёсткая рамка

«Тайный покупатель» — легитимная маркетинговая практика. Условия: **один реальный разведочный контакт под контролем Повелительницы**, без массовой фабрикации личностей, **без обхода капчи/антибота**. ToS площадок агрессивно не нарушаем. Записывается в методичку как граница.

## Что НЕ делаем в MVP

- Автономную регистрацию и присутствие на живом эфире в реальном времени.
- Массовое создание личностей / ботоводство.
- Обход капчи и антибот-защит.
- Планировщик (routines/cron) — позже.

## Открытые вопросы (к старту сборки)

1. **Конкретный подопытный №1** — выбрать имя из дежурного списка `ai-competitors` (прямой вайб-кодинг/ИИ-конкурент).
2. **Scout-почта** — какой провайдер заводим (Gmail/Яндекс/прочее), включаем IMAP, кладём креды в `.env`.
3. **Scout TG-аккаунт** — отдельный номер/аккаунт под Telegram-воронки; кто его держит.
4. **Где живёт `funnel-analyst`** — в основном оркестре (`.claude/agents/`) рядом с остальными субагентами.

## Порядок сборки (когда дадим старт коду)

1. Методичка «Разведка воронки» (чек-листы по слоям + ToS-граница) — каркас, на который опираются остальные.
2. Субагент `funnel-analyst` + подключение в маршрутизацию `agent-assistant/CLAUDE.md`.
3. Трекер-шаблон `_state.md` под конкурента.
4. `scout_inbox.py` (IMAP-сборщик) — когда заведём scout-почту.
5. Боевой прогон на выбранном конкуренте.
