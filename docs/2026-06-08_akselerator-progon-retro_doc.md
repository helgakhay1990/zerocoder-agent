# Ретро: контентный сквозной прогон оркестра — эфир ИИ-Акселератора

**Дата прогона:** 2026-06-08 · **Ветка:** контентный конвейер (концепция → лендинг → анонсы).
**Зачем:** реальный прогон субагентов на живой задаче (продающий эфир Акселератора 19.06). Заполняет шаблон ретро из `docs/2026-06-04_demo-progon-retro_doc.md` боевыми данными. Материал для сдачи ДЗ.

## Что прогнали

Полный контентный конвейер вебинара, три субагента через главного оркестратора:

| Этап | Субагент | Артефакт |
|---|---|---|
| Концепция | `launch-strategist` (в контексте) | `research/2026-06_webinar-ai-accelerator_concept.md` |
| Структура лендинга | `landing-architect` (по референсу Пименова) | `research/2026-06_akselerator-webinar_landing.md` |
| Анонсы (3 волны) | `announce-writer` | `research/2026-06_akselerator-webinar_announce.md` |

Вход — фактура 08.06 (маркетинговый бриф + транскрипция установочного звонка + присланный файл сегментов ЦА).

## Что сработало

- **Дисциплина «не выдумывать» держится.** `announce-writer` взял регалии Кирилла строго из `speakers.md` (Forbes, РБК, книга, Setters Media, 400 000 аудитории) — проверка подтвердила, ничего не сочинено. `landing-architect` не выдумал кейсы (cases.md пуст) и цены тарифов — вынес в открытые вопросы.
- **Честные открытые вопросы.** Субагенты сами разметили пробелы: лендинг — 13, анонсы — 5. Система не закрывает дыры догадками.
- **Работа по референсу.** `landing-architect` реплицировал логику лендинга Пименова (захват → ценность → программа → спикер → тарифы → для кого → доверие → CTA → глоссарий), наполнив фактурой Акселератора.
- **Субагент поймал устаревший источник правды.** `landing-architect` справедливо упёрся в то, что `zerocoder-facts.md` запрещает Акселератор («не продаётся») — система не стала молча нарушать правило, а подсветила конфликт.

## Что не сработало / где буксовали

- **`announce-writer` нарушил языковые правила вопреки самопроверке.** В тексте остались буква Ё («жёсткая», «придёшь») и латиница («CEO», «Setters A-List»), хотя в самоотчёте субагент написал «без латиницы и без ё». Чистил оркестратор вручную. → правило не было в источниках правды, субагент действовал по памяти.
- **Устаревший факт-файл затормозил лендинг.** `zerocoder-facts.md` держал Акселератор в статусе «не продаётся» — субагент корректно, но вынужденно ушёл в оговорки. Источник правды отстал от реальности.
- **Расхождение тона.** ToV предписывает «вы», бриф запуска требует «ты» — субагенты не знали, что приоритетнее, и вынесли в открытый вопрос (правильно, но правило стоило иметь заранее).
- **Контент собран частично вручную.** Содержательную концепцию Ольга и оркестратор собрали вне оркестра, затем свели в него — то есть прогон был не «с нуля через launch-strategist», а консолидацией. Чистый сквозной прогон с первого субагента — на следующий раз.

## Находки (08.06)

- Самопроверка субагента по чек-листу **не гарантирует** соблюдения языковых правил, если правила нет в источнике правды — нужно вынести правило в `tone-of-voice.md` (сделано) и/или зашить в `editor` финальным проходом.
- Источники правды устаревают быстрее, чем идёт работа: при запуске нового продукта факт-файл надо обновлять ПЕРВЫМ шагом, иначе субагенты спотыкаются.
- Конфликт «глобальный ToV vs бриф запуска» по обращению — разрешается приоритетом брифа для конкретного запуска (задокументировано).

## Решения и что уже починено сегодня

- ✅ **`knowledge-base/tone-of-voice.md`** — добавлен раздел «Языковые ограничения»: без латиницы в публичных текстах (AI→ИИ, «AI Native»→«архитектор цифровой команды»/«ИИ-мышление»), без буквы Ё, англицизмы→русские синонимы. Плюс правило override обращения брифом запуска.
- ✅ **`knowledge-base/zerocoder-facts.md`** — Акселератор и Буткемп выведены из «не продаётся» (§ 3.7/3.8 наполнены фактурой; снят запрет для субагентов, с оговоркой не выдумывать цену/состав).
- ⏭️ **Следующее:** прогнать финальные тексты через `editor` (он ловит латиницу/Ё по обновлённому ToV); решить «ты/вы» по Акселератору финально; второй сквозной прогон для метрики «правки 1-й vs 2-й».

## Дополнение: прогон methodist + editor (08.06, вечер)

Конвейер достроен до конца: `methodist` (презентация эфира — 30 слайдов + гайд участника) → `editor` (финальная редактура). Артефакты: `research/2026-06_akselerator-webinar_deck.md`, `..._guide.md` (+ docx). Итого в прогоне — **5 субагентов**: launch-strategist (контекст) + landing-architect + announce-writer + methodist + editor.

**Новые находки:**
- **Субагент резюмирует вместо выдачи материала.** `methodist` на первый запрос вернул отчёт «что собрано», а не сам текст. Понадобился повторный запуск с явным «верни полный текст, не резюме». → урок: при сборе артефакта явно требовать дословную выдачу (перекликается с архитектурой «субагент не пишет файл, отдаёт текст»).
- **Языковое правило теперь работает на двух уровнях.** `methodist` снова занёс латиницу (CEO, A-List, VPN) и Ё — но `editor`, читая обновлённый `tone-of-voice.md`, их вычистил (CEO→генеральный директор, VPN→сервисы обхода, A-List→рейтинг визионеров, фаундер→основатель). **Фикс ToV подтверждён в деле.**
- **editor сам верифицировал регалии.** Заявления слайда 28 (МФТИ, исследование с вузами, 10,8 трлн ₽) editor сверил со `speakers.md` и подтвердил — не выдумка methodist, а реальные данные. Дисциплина проверки работает.
- **editor не идеален:** пропустил «Q&A» (латиница) — дочистил оркестратор (→ «вопросы-ответы»). То есть финальный человеческий/оркестраторский проход всё ещё нужен.

**Вывод по прогону:** двухуровневая защита (правило в источнике правды + `editor` на выходе) ловит большинство нарушений; остаточный контроль за оркестратором сохраняется. Полный pipeline вебинара (концепция → лендинг → анонсы → презентация → гайд → редактура) прогнан через оркестр — цельный материал для сдачи ДЗ.
