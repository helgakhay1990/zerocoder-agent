# Концепция вебинара 3 июня — OpenClaw

## Вводные

- **Дата:** 3 июня 2026
- **Тип:** вебинар по OpenClaw, по образцу [zerocoder.ru/assistant-openclaw](https://zerocoder.ru/assistant-openclaw)
- **Продукт в продаже:** один — [«Вайб-кодинг с OpenClaw»](https://zerocoder.ru/vaib-coding-s-openclaw)

### Что не сработало на прошлом эфире
- Часто всплывало сравнение с Claude Code → сбивало слушателей
- В продаже было два новых продукта → размывался фокус

Из этого — фиксируем два правила:
1. Про сравнение с Claude Code говорим **один раз в начале практической части**, отдельным блоком, и дальше не возвращаемся
2. В продаже только один продукт — фокус не уплывает

## Задачи практикума

Три цели, которые нужно закрыть **внутри одних и тех же 30 минут**:

1. **Вау-эффект через результат**, а не через интерфейс. Зритель видит готовый агент в действии, а не клики по экрану.
2. **Чёткое отличие от Claude Code** одной формулировкой и тем, что показано. Без долгих сравнений.
3. **Дефицит навыка** — кейсы выглядят просто, но без инструкции не повторить. Это рождает желание прийти учиться.

Ключевой приём: **не разбираем механику сборки в эфире**. Показываем готовое + короткую вставку «вот шаги, по которым это собрано» (список из 4–5 пунктов). Это снимает соблазн уходить в сравнение с Claude Code.

## Формат — «Один день — четыре агента»

Четыре истории под четыре сегмента ЦА. Каждая ~6 минут, по одинаковой структуре:

- 1 мин — портрет героя и боль
- 2,5 мин — демо записанного результата
- 1,5 мин — слайд «как это собрано»
- 1 мин — «что человек делает теперь»

Спикер на камеру, без слайдов:

> Чтобы сразу снять вопрос, который вы будете задавать: чем это отличается от Claude Code.
> Claude Code работает, пока вы с ним разговариваете. Это умный собеседник: открыли окно, поставили задачу, получили результат, закрыли — он закончил.

## Связка с оффером (4:30)

Не продажа, а мост. Продажа идёт следующим блоком эфира.

Что нужно сказать:

1. Сведение четырёх историй: «Четыре разных человека, четыре разных задачи. У каждого один и тот же навык — собирать под себя ИИ-агента, который делает работу. Этот навык — то, чему мы учим.»
2. Ограничение: «Не обещаю, что вы соберёте такого агента за вечер. Аня собрала за две недели, Сергей — за три, Марина — за полтора месяца. Это работа, но работа понятная.»
3. Переход: «Сейчас покажу программу и кто из вас точно справится, а кто пока нет.»

После — спикер уходит в продающий блок про курс.

## Разница OpenClaw vs Claude Code — для слайда и спикера

### Главная ось разграничения

**Claude Code работает, пока ты с ним разговариваешь. OpenClaw работает, когда тебя рядом нет.**

Эта ось НЕ зависит от технической базы пользователя — Claude Code используют и непрограммисты для работы с документами, текстами, поиском. Поэтому разграничение «для технарей / для всех» — ошибочное. Настоящее разграничение — про **режим работы**.

### Развёрнуто

**Claude Code — это собеседник.** Открыл окно — поставил задачу — получил результат. Закрыл — он закончил. Главное:
- Работа в режиме диалога, прямо сейчас
- Результат — один конкретный артефакт
- Когда не пишешь — он ничего не делает
- Умный ассистент на смене

**OpenClaw — это сборка сотрудника, который остаётся.** Ты не получаешь ответ на сегодняшнюю задачу. Ты собираешь работника, который потом сам делает эту задачу каждый день. Главное:
- Работа без тебя, по расписанию или по событию
- Результат — постоянно работающий процесс
- Когда не пишешь — он всё равно работает
- Сотрудник со своей рабочей сменой, на которой тебя нет

### Сравнительная таблица

| | Claude Code | OpenClaw |
|---|---|---|
| Режим | Диалог здесь и сейчас | Автономная работа в фоне |
| Когда работает | Когда ты открыл окно | По расписанию или событию, без тебя |
| Что получаешь | Конкретный артефакт по запросу | Процесс, который идёт сам |
| Что остаётся, когда закрыл | Ничего не работает дальше | Работник продолжает работать |
| Чему служит | Ускорить твою работу | Снять с тебя работу совсем |

### Формулировки для спикера

**Короткая (для слайда):**
«Claude Code работает, пока ты с ним разговариваешь. OpenClaw работает, когда тебя рядом нет.»

**Средняя (20 секунд):**
«Claude Code — это умный собеседник. Открыли окно, поставили задачу, получили результат. Закрыли — он закончил. OpenClaw — другое. Вы один раз настраиваете работника, и он делает эту задачу каждый день сам. Пока вы спите, пока в отпуске, пока заняты другим. Один ускоряет вашу работу. Другой её забирает.»

**Длинная (минута, для блока введения):**
«Перед практикой сниму вопрос, который точно возникнет — чем OpenClaw отличается от Claude Code. Это два разных режима жизни. Claude Code — режим диалога. Вы пришли — он работает, вы ушли — он не работает. Это умный ассистент на смене. И многие из вас, возможно, уже им пользуетесь — для работы с документами, текстами, поиском. Это нормально и хорошо.
OpenClaw делает другое. Вы один раз объясняете, что должно происходить, и дальше это происходит само. Без вашего открытого окна. Каждый день. По расписанию. Реагируя на события. Это уже не помощник на смене — это сотрудник, который у вас есть.
Простая разница: Claude Code — собеседник. OpenClaw — работник. И вам сегодня — про второе.»

## Что подготовить к эфиру

### Видео-материалы — 4 записи экрана, по 90–150 секунд

1. Разборщик заявок — ускоренная съёмка «час за 30 секунд»
2. Семейный помощник — фото холодильника → меню → список покупок
3. Онбоардер ремонтной студии — диалог в чате с клиентом
4. Нутрициолог — диалог в WhatsApp + дашборд для Марины

### Слайды — 9 штук

- 1 разграничитель «Claude Code / OpenClaw»
- 4 портретных слайда героев (с буллетами «до»)
- 4 слайда «из чего состоит» / «что изменилось» / «экономика»

### Реквизит для убедительности

- В кейсе 3 — реальные цифры от знакомого предпринимателя (можно у любого выпускника, переименовать героя)
- В кейсе 4 — реальный выпускник декабрьского потока, желательно с видео-цитатой 15 секунд

## Что осталось решить, чтобы идти дальше

1. **Герои настоящие или композитные?** Если есть реальные выпускники под кейсы 1, 3 и 4 — даём настоящие имена и фото, это сильнее. Если нет — пишем «один из участников» без имени.
2. **Цифры по Марине и Сергею — из реальных кейсов или собрать у выпускников?** От этого зависит, насколько уверенно их можно произносить в эфире.
3. **Кто спикер?** Если Михаил — подстраиваем реплики под его манеру. Если другой — нужен референс одной его прошлой речи.

После ответов на эти вопросы — следующий слой: тексты слайдов и реплики спикера дословно.
