## 1. План аудита и что собрано через Playwright MCP

Playwright MCP позволяет загрузить страницу в реальном браузере и выполнить произвольный JS, поэтому через него реально снять почти весь чек-лист SEO-аудита одной страницы:

| Блок | Что собирается через Playwright |
|---|---|
| Мета-теги | `title`, `description`, `robots`, `viewport`, `og:*`, `twitter:*`, кастомные мета — через `document.querySelectorAll('meta')` |
| Индексация | `<meta name="robots">`, `<link rel="canonical">`, `lang`, `hreflang` |
| Структура | Все `<h1>`–`<h6>`, иерархия, дубли |
| Контент | `innerText`, объём в словах, плотность |
| Изображения | `src`, `alt`, `loading`, реальные размеры (`naturalWidth/Height`) |
| Ссылки | Все `<a href>`: внутренние/внешние, `rel`, `nofollow`, `target` |
| Структурированные данные | `<script type="application/ld+json">` |
| Производительность | `performance.getEntriesByType('navigation')` и `'resource'`: TTFB, DOMContentLoaded, Load, веса по типам |
| DOM | Количество узлов, скрипты, стили, iframe |
| Консоль | `browser_console_messages` — JS-ошибки, влияющие на интерактив |
| Мобильная адаптивность | `browser_resize(375×812)` + проверка overflow и размеров тач-зон |
| Сеть | `browser_network_requests` — все запросы со статусами |

Что **нельзя** собрать через Playwright (взято через `curl`):
- HTTP-заголовки оригинального ответа (Cache-Control, HSTS, Content-Encoding, X-Robots-Tag)
- `robots.txt`, `sitemap.xml` — отдельные ресурсы
- Цепочки редиректов (http→https, www→non-www)

## 3. Критические проблемы

### 3.1. Страница закрыта от индексации
```html
<meta name="robots" content="noindex">
```
И при этом URL **не указан** в `https://zerocoder.ru/sitemap.xml` (474 URL в sitemap, искомого среди них нет).

**Что это значит:** страница в принципе не должна получать органический трафик из Google/Яндекса. Если задача лендинга — собирать только платный трафик и переходы из писем/телеги, всё нормально. Если планировался органический подгон — конфигурация полностью его блокирует.

**Решение (если нужен органический трафик):**
1. Убрать `<meta name="robots" content="noindex">`.
2. Добавить URL в `sitemap.xml`.
3. Проверить, не закрыт ли путь в `robots.txt` (на 2026-05-18 — открыт).

### 3.2. Два тега `<h1>` на странице
```
H1 #1: «Программа персонального менторства для предпринимателей»
H1 #2: «ИИ-трансформация бизнеса с Кириллом Пшинником»
```
Поисковики ожидают **один** `<h1>` — главный заголовок. Второй стоит понизить до `<h2>`, а текущие 16 `<h2>` пересобрать с использованием `<h3>` для дочерних блоков (сейчас вся иерархия плоская: 2×H1 + 16×H2, ни одного H3–H6).

### 3.3. У 20 из 20 изображений пустой `alt=""`
Включая: логотипы партнёров, баннеры этапов программы, иконки соцсетей. Это:
- ломает доступность (screen readers);
- лишает страницу шансов на трафик из Google Картинок и Яндекс.Картинок;
- по нормам ФЗ № 419-ФЗ (доступность для людей с инвалидностью) — формально нарушение.

**Что писать:**
- Логотипы вузов/партнёров → название организации (например, «Университет Иннополис»).
- Скриншоты/баннеры → краткое содержательное описание.
- Чисто декоративные SVG-иконки → можно оставить `alt=""`, но только осознанно.

### 3.4. Нет JSON-LD структурированных данных
Ни одного блока `<script type="application/ld+json">`. Для лендинга платной программы критически отсутствуют:
- **Course** (название, провайдер, описание, цена, длительность) — даёт rich snippet в Google.
- **Organization** + **Person** (Кирилл Пшинник как `instructor`) — авторитет автора.
- **BreadcrumbList** — навигационные «хлебные крошки» в выдаче.
- **FAQPage** — если внизу будет блок FAQ.

Без этого Google не понимает, что страница продаёт обучение, и не подсветит её специальной карточкой даже на брендовых запросах.

## 4. Мета-теги — детально

| Тег | Значение | Оценка |
|---|---|---|
| `<title>` | «ИИ-трансформация бизнеса с Кириллом Пшинником» (45 симв.) | 🟢 оптимально (50–60 — потолок). Можно добавить бренд: «… — Зерокодер» |
| `description` | 157 симв. | 🟢 в зоне 150–160. Заканчивается на «масштабирования бизнеса.» — пограничный лимит |
| `keywords` | пусто | 🟢 не нужен, поисковики его игнорируют |
| `viewport` | `width=device-width, initial-scale=1.0` | 🟢 |
| `robots` | `noindex` | 🔴 см. 3.1 |
| `og:url` | совпадает с canonical | 🟢 |
| `og:title` | дубль title | 🟢 |
| `og:description` | дубль description | 🟢 |
| `og:type` | `website` | 🟡 лучше `product` или `course` (через JSON-LD) |
| `og:image` | 504×неизв., PNG | 🟡 для соцсетей нужно 1200×630, иначе картинка обрежется/растянется |
| `twitter:*` | отсутствуют | 🟡 добавить `twitter:card`, `twitter:title`, `twitter:image` |
| `<html lang>` | **не задан** | 🟡 поставить `lang="ru"` |
| `<link rel="canonical">` | корректный self-canonical | 🟢 |
| `hreflang` | отсутствует | 🟢 не нужно, если только русский |

## 5. Структура заголовков

```
H1 (2):
  1. Программа персонального менторства для предпринимателей

H2 (16): «Что такое ИИ-трансформация бизнеса?», «Методика Кирилла Пшинника…»,
         «Авторская методика…», «Трансформация: было → стало», «Главный итог»,
         «Это про переход…», «Что получает предприниматель», «Для кого…»,
         «Как всё будет?», «Калькулятор эффекта…», «Варианты участия:»,
         «Закрытое бизнес-сообщество…», «Ваш результат после прохождения…»,
         «Проводим исследования…», «Наши премии и рейтинги»

H3–H6: 0
```

**Что чинить:**
- Один H1.
- Внутри тарифов («Бизнес», «Ультима»), этапов программы (Этап 1–7), пунктов «для кого» — это естественные H3. Сейчас всё это либо обычный текст, либо элементы списка, либо такие же H2.

## 6. Контент

- **Объём:** ~1 817 слов — нормально для лендинга такой длины.
- **Главная сущность:** «ИИ-трансформация бизнеса», «Кирилл Пшинник», «менторство», «автоматизация», «зерокод», «нейроклуб» — представлены равномерно.
- **Призывы к действию:** «Узнать подробнее», «Зайти в программу», «Пройти собеседование», «Оставить заявку», «Записаться на собеседование» — обилие разных формулировок, **0 форм** на странице (`<form>` = 0). Все CTA ведут на сторонние виджеты GetCourse и попапы.
- **Дубли:** «Закрытое бизнес-сообщество предпринимателей» повторяется в двух блоках почти дословно — для пользователя ок, для SEO нейтрально.

## 13. HTTP-заголовки, robots.txt, sitemap

**Ответ страницы:**
```
HTTP/2 200
server: ddos-guard
content-type: text/html; charset=UTF-8
content-encoding: gzip          ← включено, хорошо
last-modified: Mon, 18 May 2026 08:52:57 GMT
etag: …
x-frame-options: SAMEORIGIN
```
Отсутствуют: `Cache-Control`, `Strict-Transport-Security` (HSTS), `X-Robots-Tag`.

**Редиректы:**
- `http://zerocoder.ru/…` → 301 на HTTPS ✅
- `https://www.zerocoder.ru/…` → 301 на non-www ✅

**robots.txt:**
- Корректно отдаётся, `Sitemap:` заявлен (3 файла: основной, store, feeds).
- Аудируемый URL **не** запрещён в `robots.txt` — но запрещён мета-тегом `noindex` в самой странице.

**sitemap.xml:**
- 474 URL.
- **Аудируемого URL в sitemap нет.** Согласовано с `noindex`, но если решите открыть страницу для индекса — нужно ещё и добавить её сюда.
