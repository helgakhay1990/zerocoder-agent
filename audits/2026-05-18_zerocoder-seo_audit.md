# SEO-аудит zerocoder.ru

**Дата проведения:** 2026-05-18
**URL аудита:** https://zerocoder.ru/
**Инструмент сбора данных:** MCP Playwright (headless Chromium) + WebFetch для robots.txt и sitemap.xml
**Тип аудита:** технический и контентный аудит главной страницы; проверка индексных файлов сайта

---

## 1. Краткие выводы (TL;DR)

Площадка построена на конструкторе Tilda. Базовая разметка и инфраструктура для индексации в норме: HTTP-200, корректный canonical, мобильный viewport, sitemap из 582 URL и robots.txt со Sitemap-директивами. Но при детальной проверке вылезает несколько серьёзных дефектов, которые ограничивают видимость в поиске и просадку поведенческих метрик.

**Критические находки**:
- **154 изображения из 154 без атрибута `alt`** — провал по image SEO и доступности.
- **0 структурированных данных JSON-LD** — нет ни `Organization`, ни `WebSite`, ни `Course`, ни `BreadcrumbList`. Это лишает сайт сниппетов, knowledge panel и блоков в Яндекс/Google.
- **`<html lang>` не задан** — поисковики и переводчики не получают явного сигнала о языке.
- **Title 115 символов и description 232 символа** — оба превышают рекомендуемую длину; в SERP усекутся.
- **Twitter Card отсутствует** — превью в X/Twitter будет дефолтным.
- **0 изображений с `loading="lazy"`** — все 154 картинки грузятся одновременно, декодируемый размер страницы 6.5 МБ.

**Что в норме**:
- robots.txt валиден, есть `Clean-param`, прописаны три sitemap.
- sitemap.xml корректен, 582 URL с `lastmod`.
- canonical соответствует URL, нет цепочки редиректов.
- DOM ~5 200 узлов — для главной такого объёма приемлемо.
- `noscript`-блоки (10 шт.) — есть резервный контент для краулеров без JS.

## 2. План аудита

### 2.1. Что собирается с помощью MCP Playwright
| Источник | Что извлекается |
|---|---|
| `browser_navigate` | финальный URL, HTTP-статус, заголовок страницы |
| `browser_snapshot` | accessibility-дерево (структура контента) |
| `browser_evaluate` (JS в DOM) | title, description, robots, canonical, hreflang, OG/Twitter, H1–H6, alt у `<img>`, ссылки внутренние/внешние/nofollow, JSON-LD, performance API, размер DOM, ключевые слова в тексте |
| `browser_network_requests` | общее число запросов, статусы, размеры, медленные ресурсы, внешние CDN |
| `browser_console_messages` | ошибки и warning'и JS |
| `browser_take_screenshot` | визуальная фиксация (опционально) |

### 2.2. Что собирается отдельно (WebFetch)
- `robots.txt` — директивы, sitemap-ссылки, clean-param.
- `sitemap.xml` — тип (index / urlset), число URL, теги `lastmod`/`changefreq`/`priority`.

### 2.3. Чего собрать **нельзя** в пределах Playwright (нужны внешние сервисы)
- Реальные показатели Core Web Vitals по живому трафику (нужен PageSpeed Insights / CrUX).
- Позиции в выдаче, доля показов, CTR — Яндекс.Вебмастер, Google Search Console.
- Внешний ссылочный профиль — Ahrefs / Megaindex / Букварикс.
- История индексации — Яндекс.Вебмастер.

### 3.9. robots.txt

Файл доступен по `https://zerocoder.ru/robots.txt`, отдаётся текстом.

**Структура**:
- Два блока: `User-agent: *` и `User-agent: Yandex` с идентичным набором правил.
- ~400+ директив `Disallow`. Это специфика Tilda-проекта с большим числом одностраничников: служебные адреса вебинаров, акций, dod-страниц, корзин, попапов, регистраций.
- `Allow` — нет.
- `Clean-param: ref&cl_uid&amp&gcpc&param&etext&sub1&erid&utm_source&utm_medium&utm_campaign&transaction_id` — корректно настроена очистка от UTM, Яндекс.Маркет erid, реферал-параметров.
- Три карты сайта:
  - `https://zerocoder.ru/sitemap.xml`
  - `https://zerocoder.ru/sitemap-store.xml`
  - `https://zerocoder.ru/sitemap-feeds.xml`
- `Crawl-delay`, `Host`, `Request-rate` — нет (для Яндекса `Host` в современных версиях больше не требуется).

**Замечания**:
- Внимательно проверить, не закрыты ли в `Disallow` страницы, на которые направлен платный трафик и которые должны индексироваться. Большой список выглядит как «закрыть всё прошлогоднее», но среди 400+ строк легко мог затесаться актуальный лендинг.
- Дублирование блока для `Yandex` идентично `*` — современная практика разрешает оставить только `*`. Сейчас это просто дублирование, не вред.

## 4. Приоритизированный список рекомендаций

### 🔴 Критично — сделать в первую очередь

1. **Поставить корректные `alt` всем картинкам в Tilda.** Минимум — для смысловых: логотипы партнёров, обложки программ, скриншоты интерфейсов, лица спикеров. Для чисто декоративных — пустой `alt=""` оставить осознанно, а не по умолчанию.
2. **Добавить JSON-LD блоки.** В первую очередь — `Organization` (с логотипом, адресом, телефоном, соцсетями) и `WebSite` со `SearchAction`. Вторым шагом — `Event` для регулярных вебинаров и `Course` для программ. Tilda позволяет вставлять JSON-LD через блок T123 (HTML).
3. **Прописать `<html lang="ru">`.** Через Tilda Pro делается в настройках сайта или вставкой в head.
4. **Сократить `<title>` до 60–65 символов** и `<meta description>` до 150–155. Текущие версии усекутся в SERP, потеряют CTR.
5. **Добавить блок Twitter Card** (минимум: `twitter:card=summary_large_image`, `twitter:title`, `twitter:description`, `twitter:image`).

### 🟡 Высокий приоритет

6. **Включить `loading="lazy"` для всех изображений ниже первого экрана.** Это снизит TTI и уменьшит расход трафика — особенно на мобильных.
7. **Проверить блок `Disallow` в robots.txt** и убрать оттуда лендинги, на которые идёт активный трафик и которые должны индексироваться.
8. **Подровнять `og:url` и canonical** — сейчас один со слэшем в конце, второй без. Привести к единому виду.
9. **Поработать с ключевыми словами в видимом тексте**: добавить «зерокодинг», «no-code», «нейросеть» (ед.ч.) органически в заголовки и подводки. Сейчас бренд-слово встречается всего 3 раза.
10. **Переосмыслить H2-структуру**: вместо `«Что ты получишь»`, `«Как все будет?»` использовать ключевые формулировки: `«Что входит в обучение зерокодингу»`, `«Как проходят онлайн-курсы по нейросетям»`.

### 🟢 Средний приоритет

11. **Перейти на sitemap-index** с разделением по разделам (курсы, акции, вебинары, новости).
12. **Удалить дублирующий блок `User-agent: Yandex`** из robots.txt, если правила полностью идентичны блоку `*`.
13. **Прописать `<meta name="robots" content="index, follow">`** явно — особенно для главной и страниц программ.
14. **Проверить корректность подключения Яндекс.Метрики** и Google Tag Manager — в перечне сетевых запросов их не видно отдельной строкой.
15. **Добавить `target="_blank"` и `rel="noopener noreferrer"`** на внешние ссылки (Forbes, RBC, Afisha, госреестр), если их сейчас нет.

### 🔵 Желательно

16. **Внедрить FAQ-разметку** на странице с программами и регулярными вебинарами.
17. **Внедрить Review/AggregateRating** для блока отзывов студентов.
18. **Прогнать страницу через PageSpeed Insights** и Яндекс.Вебмастер для получения данных Core Web Vitals по живому трафику.
19. **Аудит ссылочного профиля** (Ahrefs / Megaindex) — оценить долю токсичных ссылок и распределение по анкорам.
20. **Проверить URL с похожими slug'ами в sitemap** (`dod-0526`, `r-dod-0526`, `updates-r`, `updates-new`) на каннибализацию запросов и схожесть контента; склеить через canonical или 301.

## 6. Что аудит **не покрывает** и что стоит добавить

- **Core Web Vitals по реальному трафику** — нужны PageSpeed Insights, Search Console, CrUX.
- **Лог-анализ обхода ботами** — нужны логи веб-сервера или панель Tilda.
- **Внешний ссылочный профиль** — Ahrefs/Megaindex.
- **Реальные позиции, показы, клики** — Яндекс.Вебмастер, Google Search Console.
- **Глубокий контентный аудит** по 582 URL из sitemap — это отдельная работа: классификация (актуальный / прошедший вебинар / тестовая страница), решение по каждому (оставить / закрыть индексацию / редирект / удалить).
- **A/B-аудит тайтлов** — какой формат title даёт больше CTR.

Если нужно — могу собрать продолжение: разобрать sitemap по категориям URL и подсветить кандидатов на 301/`noindex`.
