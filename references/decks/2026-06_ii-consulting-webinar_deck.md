# Эталон структуры: образовательно-консультативный вебинар (мягкая продажа через заявку)

> **Что это:** эталонная дуга **экспертно-образовательного** продающего эфира Zerocoder (тема — «оптимизация бизнес-процессов за счёт ИИ» / ИИ-консалтинг, спикер Кирилл Пшинник, ~140 слайдов).
> **Когда взят:** 2026-06, из реального эфира.
> **Чем хорош:** показывает ВТОРОЙ тип дуги — не «крючок → демо → жёсткий оффер» (как продающий практикум OpenClaw, см. соседний эталон), а **«глубокая польза → доказательство → мягкий оффер через заявку на собеседование»**. Подходит для тем, где продукт премиальный/сложный и решение принимают через диалог, а не импульс.
> **Важно:** ниже — СТРУКТУРА и паттерны, не дословный контент. Конкретные цифры/кейсы/цены берутся из источников правды под конкретный запуск, не отсюда.

---

## Когда использовать эту дугу (а не продающий практикум)

- Продукт дорогой/сложный, продаётся через **заявку → собеседование**, а не «оплати на эфире».
- Аудитория — собственники/руководители/эксперты, которым важна логика и доказательность, а не скорость.
- Цель эфира — не одна транзакция, а квалификация и заявки в воронку.
- Если нужен импульсный оффер с демо и дедлайном — берём продающий эталон (`2026-06_openclaw-praktikum_deck.md`).

## Карта дуги (фазы)

| Фаза | Слайды (≈) | Задача фазы |
|---|---|---|
| **0. Тех-вход** | 1–3 | Титул + «что делать, если не видно/нет звука». Снять технический стресс до контента. |
| **1. Контракт на внимание** | 4 | «План на 2 часа: вы узнаете и увидите…». Обещание ценности + интрига (⚡ разбор кейсов в конце). |
| **2. Эксперт** | 5–6 | Регалии спикера + подписка на канал. Доверие до контента. |
| **3. Первые крючки удержания** | 7–9 | Подарок «за тех, кто останется до конца» + вовлечение в чат («напишите 7»). Закладывает досматриваемость. |
| **4. Картина мира / тренд** | 10–12 | Почему это важно именно сейчас (сдвиг рынка, новая профессия). Поднимает ставку. |
| **5. Что это за профессия/услуга** | 13–19 | Определения, отличия, «вход для не-технарей». Снимает барьер «я не технарь». |
| **6. Проблематизация на данных** | 20–27 | «95% хотят, но не знают как», «90% внедрений без результата» + исследование со ссылкой. Боль + авторитет источника. |
| **7. Ключевая идея** | 28–33 | Что реально решает (стратегическое мышление > код) + вовлечение («+ в чат, у кого есть опыт»). |
| **8. Как это работает (этапы/навыки/форматы)** | 34–42 | Пошаговый процесс, навыки, форматы занятости. Делает абстрактное осязаемым. |
| **9. Типовые кейсы (обезличенные)** | 43–48 | Проблема → решение → результат, по нишам. Узнавание «это про меня». |
| **10. Доказательство: внутренняя кухня** | 49–64 | Кейсы студентов + что компания делает сама (реальные внедрения, экономика). Самый сильный блок доверия. |
| **11. Путь к результату** | 65–87 | Как выходят на клиентов, поддержка, трудности и их отработка. Закрывает «а получится ли у меня». |
| **12. Оффер 1** | 88–110 | Вовлечение («1 в чат, кто захотел») → программа → регалии компании → состав/гарантии → CTA «заявка на собеседование, платить не нужно». |
| **13. Оффер 2 (для другого сегмента)** | 111–136 | Сегментирующий вопрос → вторая программа (премиальнее) → результаты → состав → CTA. Две двери под разные ЦА. |
| **14. Снятие возражений** | 137–140 | Подарок (повтор) + «нас часто спрашивают…» (нет опыта / сложно / заработаю ли). |
| **15. Q&A + хвост** | 141–142 | Вопросы + «кодовое слово менеджеру». Перевод тёплых в личный канал. |

## Ритм и удержание (переиспользуемые паттерны)

- **Крючок-подарок на удержание** — заявлен в начале (сл. 7) и повторён к офферам (сл. 137): «чек-лист для тех, кто до конца». Держит досматриваемость.
- **Вовлечение в чат каждые 8–12 слайдов** — «напишите N», «+ в чат, если…». Не реже; поднимает активность и сегментирует зал (опыт в бизнесе, есть дети, собственник и т.д.).
- **Сегментирующие вопросы перед офферами** — «кто собственник?», «кто хочет лично со мной?» — делят зал под нужную программу, оффер ложится точнее.
- **Проблема → Решение → Результат** — единый шаблон слайда для кейсов (фазы 9–10). Один формат, легко читать.
- **Этап → что происходит → результат этапа** — шаблон для «как это работает» (фаза 8).
- **Доказательство до оффера** — большой блок реальных внедрений (фаза 10) ставится ПЕРЕД ценой: к офферу зал уже верит.
- **Мягкий CTA** — не «оплати», а «оставь заявку на собеседование, после нажатия платить не нужно». Снимает страх, квалифицирует.
- **Две двери** — два оффера под разные сегменты (специалист / собственник), каждый со своим сегментирующим вопросом.

## Чего избегать (видно по этому эфиру)

- Длинные пачки текстовых слайдов подряд без вовлечения — между блоками теории обязательно крючок/вопрос.
- Внутренние/клиентские кейсы и цифры — только как **потенциальный результат**, с правом называть и проверкой (правило подачи кейсов — как в концепции запуска).
- Латиница/англицизмы в заголовках публичных слайдов — по tone-of-voice (AI→ИИ и т.д.).
