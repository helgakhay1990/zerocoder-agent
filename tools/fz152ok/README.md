# fz152ok — обезличивание документов одной командой

Обёртка над сервисом [fz152ok.ru](https://fz152ok.ru/) (инструмент университета «Зерокодер»).
Прогоняет документ через сервис в автоматическом браузере и кладёт на диск
обезличенный файл — чтобы безопасно отдать нейросети, подрядчику или третьим лицам.

Вся обработка (поиск ПДн + ИИ-модель) идёт **локально в браузере** — у сервиса
нет сервера, принимающего файлы. Скрипт лишь автоматизирует клики, используя
установленный в системе Google Chrome.

## Установка

Один раз:

```bash
cd tools/fz152ok
npm install
```

Отдельно ставить браузер не нужно — используется системный Google Chrome
(`/Applications/Google Chrome.app`). Если Chrome стоит в другом месте — поправь
константу `CHROME_MAC` в начале `obezlichit.mjs`.

## Использование

```bash
# просто обезличить (для передачи в нейросеть, без восстановления)
node obezlichit.mjs путь/к/документу.docx

# с восстановлением: получить ещё и ключ-карту под паролем,
# чтобы позже вернуть оригинальные данные на место
node obezlichit.mjs документ.docx --restore

# куда класть результат (по умолчанию — рядом с исходником)
node obezlichit.mjs документ.docx --out ./out
```

### Флаги

| Флаг | Что делает |
|------|-----------|
| `--restore` | режим «с восстановлением»: кроме файла сохраняет зашифрованную ключ-карту |
| `--password <...>` | пароль для ключ-карты (иначе генерируется случайный и печатается в консоль) |
| `--out <папка>` | куда сохранять результат |
| `--headful` | показать окно браузера (по умолчанию фоновый режим) |
| `--timeout <сек>` | ожидание распознавания, по умолчанию 240 |

## Форматы

`docx` · `doc` · `xlsx` · `pdf` · `txt` · `md`. Формат результата совпадает с исходным.

## Что важно знать

- **Первый запуск дольше.** Сервис один раз качает ИИ-модель (~170 МБ) — дальше
  она берётся из кэша браузера, и прогоны идут быстрее.
- **Спец-категории ПДн (ст. 10 152-ФЗ)** — национальность, религия, политвзгляды —
  автоматически не находятся. Если такие данные есть, проверь глазами.
- **Автопоиск не стопроцентный.** Правила + ИИ ловят подавляющее большинство ПДн,
  но редкий формат может ускользнуть. Для чувствительных документов сверься с
  результатом (в режиме `--headful` виден предпросмотр).
- **Пароль от ключ-карты нигде не хранится.** Потеряешь пароль — восстановить
  оригинал из ключ-карты нельзя.
- При ошибке скрипт сохраняет скриншот состояния (`fz152ok_error.png`) в папку вывода.

## Восстановление данных

Обратная замена (ключ-карта + пароль → оригинал) делается вручную на сайте:
вкладка **«Восстановить»**. Автоматизация восстановления в скрипт пока не заложена —
добавим, если понадобится поток.

## Проверка

Скрипт протестирован end-to-end на договоре с выдуманными ПДн: распознаёт ФИО,
телефон, e-mail, паспорт, ИНН, СНИЛС, адрес, банковский счёт, организацию;
оба режима кладут файл на диск, ключ-карта шифруется (AES, Web Crypto).
